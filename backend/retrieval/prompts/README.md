# Prompts 管理

本文件夹用于管理所有 LLM 提示词（Prompts）。

## 文件结构

```
prompts/
├── README.md                  # 本文档
├── extraction.md              # 知识图谱提取提示词
├── document_topic.md          # 文档主题提取提示词
├── extraction_prompts.py      # Python 格式提示词（已废弃）
└── qa_prompts.py              # QA 提示词
```

## 使用方式

### 1. Markdown 格式（推荐）

Markdown 文件可以直接被非开发人员编辑，使用占位符 `{变量名}` 来定义模板变量。

**示例**：
```markdown
# 角色
你是知识图谱提取专家。

# 待提取文本
{text}
```

**Python 代码调用**：
```python
from prompts.prompt_loader import load_prompt

# 加载提示词
prompt = load_prompt('extraction.md', text="待提取的文本内容")
```

### 2. 修改提示词

1. 编辑对应的 `.md` 文件
2. 保存文件
3. 重启服务或重新加载模块

## 提示词设计原则

### 1. 结构化
使用 Markdown 的层级结构（`##`, `###`）组织提示词，使其易于阅读和维护。

### 2. 明确性
- 明确定义角色和任务
- 提供清晰的规则和约束
- 给出具体的示例

### 3. 可配置
- 使用占位符（如 `{text}`, `{context}`）
- 支持条件逻辑（通过 prompt_loader）

### 4. 可测试
- 每个提示词应包含示例
- 示例应覆盖常见情况和边界情况

## 提示词模板

创建新提示词时，请使用以下模板：

```markdown
# [提示词名称]

## 角色
你是[角色定义]。

## 任务
[任务描述]

## 规则
- 规则1
- 规则2
- 规则3

## 输出格式
[期望的输出格式]

## 示例
**输入**：[示例输入]
**输出**：[示例输出]

## 待处理内容
{text}
```
